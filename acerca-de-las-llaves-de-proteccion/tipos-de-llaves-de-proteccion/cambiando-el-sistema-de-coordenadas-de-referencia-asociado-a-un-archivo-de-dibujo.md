# Cambiando el Sistema de Coordenadas de Referencia asociado a un archivo de dibujo

Si tenemos un archivo de dibujo que tiene asociado como sistema de coordenadas de referencia uno incorrecto \(por ejemplo, el archivo de dibujo tiene asociado el sistema de coordenadas de referencia _WGS 84 / UTM Zone 30N_ cuando en realidad debería tener asociado el _WGS 84 / UTM Zone 19N\)_, podemos solucionarlo cambiando el sistema de coordenadas de referencia asociado el archivo.

Los pasos para realizar esta tarea varían en función del formato del archivo de dibujo, ya que algunos como [.bind](cambiando-el-sistema-de-coordenadas-de-referencia-asociado-a-un-archivo-de-dibujo.md) tienen asociado en el propio archivo el sistema de coordenadas de referencia y otros como [.bin](cambiando-el-sistema-de-coordenadas-de-referencia-asociado-a-un-archivo-de-dibujo.md) almacenan el sistema de coordenadas de referencia en otro archivo con el mismo nombre que el archivo de dibujo pero con extensión [.prj](cambiando-el-sistema-de-coordenadas-de-referencia-asociado-a-un-archivo-de-dibujo.md).
